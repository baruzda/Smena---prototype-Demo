# Смена X5: интерактивный прототип

Мобильный web-first прототип подбора подработок для сотрудников ритейла. Интерфейс построен на React и Vite, использует mock-данные и работает без бэкенда.

## Запуск

```bash
npm run dev
```

Сборка для проверки:

```bash
npm run build
```

## Реализованные сценарии

### Список заданий

- Таймлайн показывает задания на 14 дней вперёд.
- Нажатие на дату плавно прокручивает список к соответствующему дню.
- У каждого дня есть демонстрационные карточки с разными ролями, адресами, оплатой, временем, расстоянием и брендами X5.
- При сохранении новой точки на карте каталог пересобирается: меняются число заданий, адреса, расстояния, часы, оплата и бренды. Данные остаются стабильными, пока локация не изменится снова.
- Для занятых дат выводится карточка основной смены сотрудника по графику `2/2`, `10:00–22:00`.
- На 6 июня добавлена mock-карточка уже принятой подработки.
- Если заданий не осталось после фильтрации, выводится «Нет подходящих смен».
- Карточка открывает детали задания с оплатой, временем, адресом и условиями. Из деталей можно записаться на смену.
- После успешной записи смена появляется в «Моих заданиях»; таб «задания на подписание» показывает честное пустое состояние.

### Прокрутка

- Верхний системный бар остаётся видимым.
- При прокрутке вниз таббар скрывается, а кнопка «на карте» переезжает на `32px` от нижнего края.
- При обратной прокрутке закреплённо появляются табы и строка фильтров; таймлайн остаётся доступным.
- После примерно двух карточек прокрутки появляется круглая кнопка «Наверх».

### Фильтры и сортировка

- Тумблер «подходит мне» включён по умолчанию и показывает только задания с соответствующим бейджем на доступные даты.
- При выключенном тумблере доступны все mock-задания, прошедшие сохранённые фильтры.
- Сортировка: «сначала ближайшие», «рекомендуемые», «выше заработок».
- Экран фильтров содержит фильтры по сети, услуге, минимальной оплате, территории и радиусу. Все выбранные фильтры применяются к выдаче после «сохранить».
- Текущую комбинацию фильтров, территории и радиуса можно сохранить как подборку. Во вкладке «избранное» есть разделы «магазины» и «подборки»: сохранённую подборку можно применить к выдаче или удалить. Раздел магазинов показывает пустое состояние, пока в прототипе нет отдельного сценария добавления магазина в избранное.

### Территория и карта

- Селектор «территория» и плавающая кнопка «на карте» открывают карту OpenStreetMap.
- Можно панорамировать и масштабировать карту, поставить точку кликом, изменить радиус и применить результат.
- Поле адреса поддерживает поиск по Enter или кнопке «найти».
- Клик по карте использует обратное геокодирование, чтобы подставить название места в поле.
- Черновой текст адреса нельзя применить до успешного поиска: в выдачу всегда попадают согласованные адрес и координаты.
- После неудачного поиска можно одним действием вернуть последнюю подтверждённую точку.
- Карта подбирает масштаб так, чтобы круг выбранного радиуса помещался над нижней шторкой; при сетевой ошибке доступна повторная загрузка.
- Новая точка меняет mock-каталог заданий; расстояния покрывают весь диапазон `1–50 км`, поэтому смена радиуса заметно меняет выдачу.
- Карта загружает Leaflet из CDN, тайлы OpenStreetMap и геокодирование Nominatim. Для этого сценарию нужен доступ к интернету.

### Настройки расписания

- Базовый экран содержит селектор «выберите дни», ручной ввод времени и пресеты доступности.
- Календарь свободных и занятых дней открывается только через селектор.
- На свободные даты можно нажать повторно, чтобы отменить выбор; занятые даты недоступны.
- Вручную изменённое время снимает выделение с чипов пресетов и остаётся ручным после повторного открытия настроек.
- Выбор конкретных дат, регулярных дней недели, времени и предпочтительной длительности применяется после «сохранить» на экране настроек и ограничивает выдачу «подходит мне».

## Данные и сохранение

- Все данные являются mock-данными июня.
- Выбранные фильтры, сохранённые подборки, радиус, территория, настройки доступности, сортировка и записи сохраняются в `localStorage` браузера.
- Запись на подработку реализована как локальный сценарий без серверной валидации, авторизации и обмена с реальным работодателем.

## Внешние ресурсы
