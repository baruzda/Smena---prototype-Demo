# E2E gap report: карточки услуг

Дата фиксации: 2026-07-22

## Исходный статус

Предыдущий запуск E2E нельзя было считать подтверждённо успешным: среда потеряла сессию вывода до финальной сводки.

При анализе `tests/app.spec.mjs` были обнаружены устаревшие селекторы:

- `.task-card`;
- `.task-card-special`;
- `.match-badge`;
- `.task-restrictions`;
- `.task-address-text`.

Текущая реализация карточек использует пространство имён `gig-task-*`. Разрыв классифицирован как **existing stale selector**, а не как регрессия архитектуры правил.

## Исправление

E2E-тесты переведены на актуальные селекторы:

| Устаревший селектор | Актуальный селектор |
| --- | --- |
| `.task-card` | `.gig-task-card` |
| `.task-card-special` | `.gig-task-card-special` |
| `.match-badge` | `.gig-task-badge` |
| `.task-restrictions` | `.gig-task-restrictions` |
| `.task-address-text` | `.gig-task-address-text` |

Дополнительно обновлены устаревшие тестовые контракты:

- тесты больше не подменяют штатное состояние приложения неполным объектом `localStorage`;
- учитывается актуальная версия онбординга и версия сохранённого состояния;
- дефолтный тумблер `Подходит мне` считается выключенным;
- проверяются текущие действия информационной карточки скрытых услуг;
- сохранённая подборка проверяется по актуальному названию, структуре и сценарию редактирования;
- используются актуальные aria-label настроек доступности и территории;
- неоднозначный поиск карточки по повторяющемуся названию ограничен конкретным элементом.

Компоненты React, DOM-контракт карточек и CSS ради совместимости со старыми тестами не изменялись.

## Защита от повторения

В `.github/workflows/card-rules-validation.yml` добавлен обязательный проход:

1. базовая валидация реестров;
2. категоризированная cross-registry валидация;
3. изолированные fixtures валидатора;
4. production-сборка;
5. полный Playwright E2E-набор.

Workflow сохраняет текстовый вывод и Playwright-диагностику как CI-артефакт. `set -o pipefail` не позволяет `tee` скрыть код ошибки тестов.
