# E2E gap report: карточки услуг

Дата фиксации: 2026-07-22

## Исходный статус

Предыдущий запуск E2E нельзя было считать подтверждённо успешным: среда потеряла сессию вывода до финальной сводки.

При анализе `tests/app.spec.mjs` были обнаружены устаревшие селекторы:

- `.task-card`;
- `.task-card-special`;
- `.match-badge`;
- `.task-restrictions`;
- `.task-address-text`.

Текущая реализация карточек использует пространство имён `gig-task-*`. Разрыв классифицирован как **existing stale selector**, а не как регрессия архитектуры правил.

## Исправление

E2E-тесты переведены на актуальные селекторы:

| Устаревший селектор | Актуальный селектор |
| --- | --- |
| `.task-card` | `.gig-task-card` |
| `.task-card-special` | `.gig-task-card-special` |
| `.match-badge` | `.gig-task-badge` |
| `.task-restrictions` | `.gig-task-restrictions` |
| `.task-address-text` | `.gig-task-address-text` |

Изменены только тестовые селекторы. Компоненты React, DOM-контракт карточек и CSS ради совместимости со старыми тестами не изменялись.

## Защита от повторения

В `.github/workflows/card-rules-validation.yml` добавлен обязательный проход:

1. базовая валидация реестров;
2. категоризированная cross-registry валидация;
3. изолированные fixtures валидатора;
4. production-сборка;
5. полный Playwright E2E-набор.

Playwright-отчёт сохраняется как CI-артефакт даже при падении тестов.

## Статус

- Причина gap определена: **закрыто**.
- Устаревшие селекторы: **исправлены**.
- Локальный потерянный вывод больше не считается доказательством прохождения.
- Финальный результат E2E определяется только GitHub Actions workflow `Card rules validation`.
